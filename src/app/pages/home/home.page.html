If you don't know where to take it from here, there are a few recommendations that could be useful:

State management

You can think about the complexity of your application and conclude if you should use a state management tool to handle the data of your application on a centralized spot.

Here are some options ordered in less-to-major boilerplate code:

  1. Akita
  2. NGXS
  3. NgRx

i18n

If you need to have your application in multiple languages, you need to think if you want server-side translations or runtime translations; if you want the first, the way to go is with the official solution , on the other hand, if you want to translate your app on runtime, the most popular solutions are transloco and ngx-translate .

Keep your styles consistent

Please keep in mind that all the code that you find on the templates is disposable; that's why the HTML is bloated with classes. In a real-world approach, to keep consistency across your application, you should consider extracting your components (buttons, inputs, modals, etc.) into Tailwind Components or use some UI components suite like (TaigaUI, Nebular, NgZorro, DevExtreme, Angular Material, etc.).
